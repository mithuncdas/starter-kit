1. Overview & Goals

Build a country-agnostic location hierarchy usable by any country in the world. The original schema (country → zone → state → district → city) is too rigid: India has zones & states, Bangladesh has divisions & upazilas, Singapore is effectively flat, USA has counties. We will collapse everything below country into a single self-referencing polymorphic table and use the staudenmeir/laravel-adjacency-list package to traverse it with recursive CTEs.

In Scope

  • Reference data tables (countries + admin areas)
  • Seeders for India, Bangladesh, USA, Singapore as proofs
  • Read-only REST API for frontend lists & cascading dropdowns
  • Eloquent Resources for clean payloads
  • Feature tests for endpoints & tree traversal

Out of Scope (Phase 1)

  • Tenant-customized areas (sales territories, delivery zones)
  • Multi-language area names
  • Admin CRUD UI — data comes from seeder only
  • Postal code import (separate task)

2. Final Schema (5 tables)

countries | +-- country_admin_structures (per-country ordered levels) | | | +-- admin_levels (global catalog: STATE, ZONE, DIVISION, ...) | +-- admin_areas (self-referencing; everything below country) | +-- postal_codes (optional, linked to lowest applicable area)

Defines, per country, which levels are used and in what order. This is the metadata that lets the same admin_areas table represent any country.

  • USA: (1,STATE), (2,COUNTY), (3,CITY)

3. Package Install

One dependency does the heavy lifting for tree traversal.

composer require staudenmeir/laravel-adjacency-list:"^1.0"

That's it — no service provider registration needed; the package auto-discovers. It adds the HasRecursiveRelationships trait we'll use on AdminArea.

Why this package? It gives us ->ancestors, ->descendants, ->bloodline, ->tree(), and materialized-path helpers using recursive CTEs (Postgres-compatible). No closure table required, no triggers to maintain, and queries stay readable.

4. Models & Migrations

Generate each model and its migration in a single command using the -m flag. Order matters so foreign keys resolve cleanly:

Project convention: All foreign keys use ->restrictOnDelete() — never cascadeOnDelete() or nullOnDelete(). The database must refuse a delete that would orphan or remove dependent rows; callers clean up explicitly.

5.4 AdminArea (the polymorphic tree)

app/Models/AdminArea.php
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\BelongsTo;
use Staudenmeir\LaravelAdjacencyList\Eloquent\HasRecursiveRelationships;

class AdminArea extends Model
{
    use HasRecursiveRelationships;

    protected $fillable = [
        'country_id', 'parent_id', 'admin_level_id', 'depth',
        'code', 'name', 'short_name',
        'latitude', 'longitude', 'timezone', 'is_active',
    ];

    protected $casts = [
        'is_active' => 'boolean',
        'latitude' => 'decimal:7',
        'longitude' => 'decimal:7',
    ];

    /**
     * Tell the recursive-relationships trait which column points to the parent.
     */
    public function getParentKeyName(): string
    {
        return 'parent_id';
    }

    public function country(): BelongsTo
    {
        return $this->belongsTo(Country::class);
    }

    public function level(): BelongsTo
    {
        return $this->belongsTo(AdminLevel::class, 'admin_level_id');
    }

    public function scopeForCountry($query, int $countryId)
    {
        return $query->where('country_id', $countryId);
    }

    public function scopeAtDepth($query, int $depth)
    {
        return $query->where('depth', $depth);
    }
}
What the trait gives us for free: $area->ancestors, $area->descendants, $area->bloodline, AdminArea::tree()->get(), AdminArea::withRecursiveQueryConstraint(...), and $area->descendantsAndSelf->pluck('name')->implode(' » ').

use App\Models\Country;
use App\Models\AdminLevel;
use App\Models\CountryAdminStructure;

class CountryAdminStructureSeeder extends Seeder
{
    public function run(): void
    {
        $structures = [
            'IN' => ['ZONE', 'STATE', 'DISTRICT', 'CITY'],
            'BD' => ['DIVISION', 'DISTRICT', 'UPAZILA', 'CITY'],
            'US' => ['STATE', 'COUNTY', 'CITY'],
            'SG' => ['REGION', 'CITY'],
        ];

        $levelIds = AdminLevel::pluck('id', 'code');

        foreach ($structures as $iso2 => $levels) {
            $country = Country::where('iso2', $iso2)->first();
            foreach ($levels as $depth => $code) {
                CountryAdminStructure::updateOrCreate(
                    ['country_id' => $country->id, 'depth' => $depth + 1],
                    ['admin_level_id' => $levelIds[$code]],
                );
            }
        }
    }
}

use App\Models\Country;
use App\Models\AdminLevel;
use App\Models\AdminArea;

class AdminAreaSeeder extends Seeder
{

        $this->seedBangladesh();

        $country = Country::where('iso2', 'IN')->first();

        $west = $this->upsertArea($country, null, $level('ZONE'), 1, 'WEST', 'Western Zone');
        $mh   = $this->upsertArea($country, $west, $level('STATE'), 2, 'MH', 'Maharashtra');
        $pune = $this->upsertArea($country, $mh, $level('DISTRICT'), 3, 'PUNE', 'Pune');
        $this->upsertArea($country, $pune, $level('CITY'), 4, 'PUNE-CITY', 'Pune City');
        $this->upsertArea($country, $pune, $level('CITY'), 4, 'PIMPRI', 'Pimpri-Chinchwad');

        $south = $this->upsertArea($country, null, $level('ZONE'), 1, 'SOUTH', 'Southern Zone');
        $ka    = $this->upsertArea($country, $south, $level('STATE'), 2, 'KA', 'Karnataka');
        $blr   = $this->upsertArea($country, $ka, $level('DISTRICT'), 3, 'BLR', 'Bangalore Urban');
        $this->upsertArea($country, $blr, $level('CITY'), 4, 'BENGALURU', 'Bengaluru');
    }

    private function seedBangladesh(): void
    {
        $country = Country::where('iso2', 'BD')->first();
        $level   = fn(string $c) => AdminLevel::where('code', $c)->value('id');

        $dhk = $this->upsertArea($country, null, $level('DIVISION'), 1, 'DHK', 'Dhaka');
        $dhkD = $this->upsertArea($country, $dhk, $level('DISTRICT'), 2, 'DHK-D', 'Dhaka District');
        $savar = $this->upsertArea($country, $dhkD, $level('UPAZILA'), 3, 'SAVAR', 'Savar');

        $ctg = $this->upsertArea($country, null, $level('DIVISION'), 1, 'CTG', 'Chattogram');
        $ctgD = $this->upsertArea($country, $ctg, $level('DISTRICT'), 2, 'CTG-D', 'Chattogram District');
        $this->upsertArea($country, $ctgD, $level('UPAZILA'), 3, 'PATIYA', 'Patiya');
    }

    private function seedUSA(): void
    {
        $country = Country::where('iso2', 'US')->first();
        $level   = fn(string $c) => AdminLevel::where('code', $c)->value('id');

        $ca = $this->upsertArea($country, null, $level('STATE'), 1, 'CA', 'California');
        $sf = $this->upsertArea($country, $ca, $level('COUNTY'), 2, 'SF', 'San Francisco County');
        $this->upsertArea($country, $sf, $level('CITY'), 3, 'SF-CITY', 'San Francisco');

    private function seedSingapore(): void
    {
        $country = Country::where('iso2', 'SG')->first();
        $level   = fn(string $c) => AdminLevel::where('code', $c)->value('id');

        $central = $this->upsertArea($country, null, $level('REGION'), 1, 'CENTRAL', 'Central Region');
        $this->upsertArea($country, $central, $level('CITY'), 2, 'SG-CITY', 'Singapore');
    }

    private function upsertArea(Country $country, ?AdminArea $parent, int $levelId,
                                int $depth, string $code, string $name): AdminArea
    {
        return AdminArea::updateOrCreate(
            [
                'country_id'     => $country->id,
                'admin_level_id' => $levelId,
                'code'           => $code,
            ],
            [
                'parent_id' => $parent?->id,
                'depth'     => $depth,
                'name'      => $name,
                'is_active' => true,
            ],
        );
    }
}

8.1 LocationController

app/Http/Controllers/Api/LocationController.php
<?php

namespace App\Http\Controllers\Api;

use App\Http\Controllers\Controller;
use App\Http\Resources\AdminAreaResource;
use App\Http\Resources\AdminAreaTreeResource;
use App\Http\Resources\CountryResource;
use App\Models\AdminArea;
use App\Models\Country;
use Illuminate\Http\Request;

class LocationController extends Controller
{
    public function countries()
    {
        return CountryResource::collection(
            Country::where('is_active', true)->orderBy('name')->get()
        );
    }

    public function structure(Country $country)
    {
        $country->load('structure.level');
        return new CountryResource($country);
    }

    /**
     * Top-level areas for a country (depth = 1).
     * Used for the first dropdown.
     */
    public function topLevel(Country $country)
    {
        return AdminAreaResource::collection(
            $country->topLevelAreas()->with('level')->orderBy('name')->get()
        );
    }

    /**
     * Children of any given area.
     * Used to cascade the next dropdown.
     */
    public function children(AdminArea $area)
    {
        return AdminAreaResource::collection(
            $area->children()->with('level')->orderBy('name')->get()
        );
    }

    /**
     * Full nested tree for a country, eager-loaded.
     * Used when the frontend wants to render the whole hierarchy at once.
     */
    public function tree(Country $country)
    {
        $tree = AdminArea::tree()
            ->where('country_id', $country->id)
            ->with('level')
            ->get()
            ->toTree();

        return AdminAreaTreeResource::collection($tree);
    }

    /**
     * Search by name across all levels for a country.
     */
    public function search(Request $request, Country $country)
    {
        $q = $request->string('q')->trim()->value();

        $results = AdminArea::forCountry($country->id)
            ->when($q !== '', fn($qq) => $qq->where('name', 'ilike', "%{$q}%"))
            ->with('level')
            ->limit(50)
            ->get();

        return AdminAreaResource::collection($results);
    }

    /**
     * Ancestor chain from root down to (and including) this area.
     * Used to build the breadcrumb on the area detail page.
     */
    public function ancestors(AdminArea $area)
    {
        return AdminAreaResource::collection(
            $area->ancestorsAndSelf()->with('level')->orderBy('depth')->get()
        );
    }
}

9. API Reference (for any frontend consumer)

Backend exposes a read-only REST API. The frontend is implemented separately and is out of scope for this plan. This section documents the contract a frontend developer needs.

9.1 Endpoint summary

MethodURLPurpose
GET /api/locations/countries List all active countries
GET /api/locations/countries/{country}/structure Country + ordered list of admin levels used (Zone → State → District …)
GET /api/locations/countries/{country}/top-level Areas at depth 1 for the country
GET /api/locations/countries/{country}/tree Full nested tree (use sparingly — can be large)
GET /api/locations/countries/{country}/search?q=text Case-insensitive name search across all levels (limit 50)
GET /api/locations/areas/{area}/children Direct children of an area (cascading drill-down)
GET /api/locations/areas/{area}/ancestors Ancestor chain (root → area), useful for breadcrumbs

9.2 Suggested call flow (master-detail drill-down)

1. On app load GET /api/locations/countries -> render country list 2. User selects country (e.g. India) GET /api/locations/countries/1/structure (cache for session) GET /api/locations/countries/1/top-level -> render that country's zones, labelled "Zone" (label comes from structure[0].label) 3. User clicks an area (e.g. Western Zone, id=10) GET /api/locations/areas/10/children (next list) GET /api/locations/areas/10/ancestors (breadcrumb) -> render children list, labelled with structure[currentDepth].label -> render breadcrumb from ancestors 4. Repeat step 3 deeper. When children is empty, you've hit a leaf node.

9.4 Common response envelope & errors

  • All success responses use the Laravel Resource envelope: { "data": … }.
  • 404 when a country/area id (or {country} ISO code) doesn't exist.
  • 422 with Laravel validation shape if a required query param is missing in future write endpoints.

9.6 Auth

These endpoints serve reference data and are public by default. If you want them protected, add the auth:sanctum middleware to the route group in routes/api.php.
